# 整体实现逻辑

## 1. 把问题拆成两层

第一层是“性质估计”：给定锂盐、温度、浓度、两种溶剂和比例，估计该配方的电导率或传输潜力、溶剂化能力、黏度、安全性和稳定性。

第二层是“组合优化”：枚举可采购溶剂的两两组合，在每个组合中搜索不同比例，再用用户设置的目标权重排序。

因此它不是简单的“根据名称查一种溶剂”，而是一个带约束的多目标优化问题。

## 2. 数据流

1. `scripts/sync_public_data.py` 下载 CALiSol-23。
2. 同一脚本调用 PubChem PUG REST，根据溶剂名称取得 CID、标准结构和基础性质。
3. RDKit 读取 SMILES，计算分子量、TPSA、LogP、氢键受体数、可旋转键等描述符。
4. 本地溶剂目录补充电池研发需要、但 PubChem 结构接口不直接提供的筛选先验，例如介电常数、黏度、供体数、闪点和粗粒度稳定性等级。
5. 所有来源与失败记录写入 `data/processed/source_metadata.json`，便于追踪。

## 3. 电导率模型

CALiSol-23 每行包含盐种类、浓度、温度、比例类型、溶剂组成和电导率 `k`。

模型输入包括：

- 温度与盐浓度；
- 38 种 CALiSol 溶剂的组成比例；
- 锂盐 one-hot 编码；
- 浓度单位和比例类型 one-hot 编码。

模型使用 Extra Trees 回归。它适合当前这种表格型、非线性、不同配方变量相互作用明显的数据。训练/测试不是随机拆行，而是按 DOI 分组：同一篇论文的相似实验不会同时出现在训练和测试中，因此指标更接近跨文献泛化能力。

模型还用 220 棵树的预测离散程度近似不确定性。离散程度越高，结果置信度越低。

## 4. 训练集外盐如何处理

CALiSol-23 不包含 LiNO3 电导率。项目另外加入了 LiNO3 实测溶解度数据：AN、乙醇、DMC、DMSO、PC 在 25–45 °C 的25条摩尔分数。系统不会把其他盐的电导率冒充成 LiNO3 电导率，而是用独立溶解度模型与传输启发式组合：

- 高介电常数帮助阴阳离子电荷分离；
- 较高 Lewis 碱性/供体数帮助 Li+ 配位；
- RDKit 极性表面积与分子尺寸构成结构极性指标；
- 黏度越低，传输代理分数通常越高；
- 加入 LiNO3 对 DMSO、glyme、DME、DOL 等溶剂类别的粗粒度亲和先验；
- 根据熔点排除室温下高比例固态组分；
- 对闪点采用偏向挥发组分的保守估计，避免线性平均造成虚假安全感。

LiNO3 的置信度不再使用固定的“42% + 每种已知溶剂6%”。新公式综合模型树间分歧、与训练样本的结构距离、局部样本密度、温度覆盖、两种组分的训练覆盖和模型/物理评分一致性。二元混合物标签仍不足，因此采用平滑置信度上限，但不会因硬截断导致大量结果显示同一个数值。

## 5. 配比是怎样找出来的

对任意溶剂 A/B，系统搜索：

```text
A:B = 10:90, 15:85, ..., 85:15, 90:10
```

每个比例计算：

- 混合介电常数；
- 对数混合黏度；
- 混合供体能力；
- 保守闪点代理；
- 溶剂化评分；
- 电导率模型值或传输代理；
- 氧化/还原稳定性先验；
- 低温与安全评分。

不满足最低闪点、最高黏度、相态或危害约束的配方被过滤。其余配方按用户权重求加权总分。最终每对溶剂只保留得分最高的比例，避免结果页被同一组合的相邻比例占满。

网页端让五个权重始终保持总和 100%，并采用从上到下的优先级。调整某一滑块时，其上方项目视为已确认值，保持不变；只有下方项目按照当前相对比例重新分配剩余权重。若上方权重与当前请求值将超过 100%，当前值会自动限制为可用的剩余额度。最后一项始终等于前四项分配后的剩余值。整数百分比通过最大余数法处理，保证显示值精确加和为 100。后端仍会再次归一化，以兼容直接调用 API 的情况。

如果严格约束下找不到足够候选，系统不会返回空白。它会重新评估完整候选空间，对违反闪点、黏度或危害限制的程度施加惩罚并降低置信度，然后把这些配方作为“放宽约束备选”补入列表。页面会显示具体违反了哪条约束，用户可以据此决定是否调整滑块或实验条件。

## 6. 如何理解 LiNO3 示例

在默认锂金属场景下，系统通常会把“强溶剂化组分 + 醚类或低黏度共溶剂”放在前列，例如 DMSO/glyme、DMSO/DME 或含 glyme 的组合。推荐比例是实验起点。

这里有两个必须注意的事实：

1. “能溶解 LiNO3”不等于“适合锂金属电池”。例如 DMSO 的溶剂化能力强，但负极界面兼容性可能成为限制。
2. 最优比例依赖目标浓度、温度、其他锂盐、添加剂和电极体系。没有对应实验标签时，不存在可信的唯一数学最优值。

合理的使用方式是取 Top 3 至 Top 5 配方，在推荐比例附近设计小型实验矩阵，例如推荐 40:60 时，同时验证 30:70、40:60、50:50。

## 7. 后续升级路线

要把“预筛选工具”升级为更接近研发级平台，最重要的不是换成更大的模型，而是加入真实标签：

- 收集 LiNO3 在单溶剂和混合溶剂中的溶解度数据；
- 统一温度、浓度和比例单位；
- 加入电化学窗口、循环效率、库仑效率和界面阻抗；
- 使用主动学习，让模型选择最值得做的下一批实验；
- 将实验结果自动回写，持续校准盐特异模型；
- 使用贝叶斯优化代替固定 5% 网格，在连续比例空间寻找 Pareto 前沿。
