# 本地训练指南

这份文档说明 Ionmix 当前模型是怎么训练的、训练完成后会得到什么、如何判断训练是否成功，以及以后怎样加入自己的实验数据。

先说结论：运行 `setup_project.ps1` 确实会把当前项目的两个模型重新训练完。但它只是“复现当前版本的训练流程”，不是让模型自动变得更聪明。模型质量取决于训练数据，尤其是有没有你关心的盐、溶剂组合、温度、浓度和实测性质。

## 1. 当前到底训练了哪些模型

项目现在有两条模型线。

### 1.1 CALiSol-23 电导率模型

文件：

```text
scripts/train_conductivity_model.py
```

输入数据：

```text
data/raw/calisol23.csv
```

输出模型：

```text
models/conductivity_model.joblib
models/training_report.json
```

训练目标：

```text
k
```

也就是 CALiSol-23 数据集中记录的电导率数值。

特征包括：

- 温度 `T`
- 浓度 `c`
- 溶剂组成比例列
- 锂盐类别
- 浓度单位
- 溶剂比例类型

当前模型：

```text
ExtraTreesRegressor
```

主要参数：

```text
n_estimators=220
min_samples_leaf=2
max_features=0.82
random_state=42
```

验证方式：

```text
GroupShuffleSplit
```

分组字段是 DOI。也就是说，同一篇文献的数据不会同时进入训练集和测试集，避免“同一来源数据泄漏”导致测试结果虚高。

当前报告大概类似：

```json
{
  "mae": 1.1942,
  "r2": 0.7406,
  "train_rows": 9915,
  "test_rows": 3320,
  "group_split": "DOI"
}
```

注意：Render 免费档默认不加载这个大模型，因为它容易超过 512 MB 内存限制。本地运行可以启用。

### 1.2 LiNO3 溶解度模型

文件：

```text
scripts/train_lino3_solubility_model.py
```

输入数据：

```text
data/lino3_solubility.csv
data/processed/solvent_catalog.csv
```

输出模型：

```text
models/lino3_solubility_model.joblib
models/lino3_solubility_report.json
```

训练目标：

```text
log10(solubility_mole_fraction)
```

也就是 LiNO3 摩尔分数溶解度的对数值。

特征包括：

- 温度
- 分子量
- TPSA
- logP
- 氢键受体数
- 可旋转键数
- sp3 碳比例
- 环数量
- 介电常数
- 黏度
- donor number

当前模型使用两部分：

```text
Ridge(alpha=10)
ExtraTreesRegressor uncertainty ensemble
```

Ridge 负责主预测，ExtraTrees 用来估计模型不确定性。

验证方式：

```text
LeaveOneGroupOut
```

这里按溶剂分组，也就是“留一个溶剂不参与训练，再预测这个溶剂”。这比随机切分更严格，因为它测试的是对新溶剂结构的外推能力。

当前数据量很小：

```text
25 条记录
5 种溶剂
25–45 °C
```

所以它能解决“完全没有 LiNO3 训练标签”的问题，但还不足以高置信度预测任意二元/三元混合溶剂。

## 2. 一键训练是不是就够了

如果你的目标是“复现当前项目”，是的，一键训练就够：

```powershell
.\scripts\setup_project.ps1
```

它会按顺序做：

1. 创建 `.venv`
2. 安装 `requirements.txt`
3. 下载/同步公开数据
4. 训练电导率模型
5. 训练 LiNO3 溶解度模型
6. 写入 `models/*.joblib` 和报告文件

但如果你的目标是“让模型更懂某个新体系”，只运行脚本不够。你必须增加新的实验数据，然后再训练。

## 3. 推荐的本地训练流程

先进入项目目录：

```powershell
cd "E:\基于机器学习的电解液关键性质预测与分子智能筛选"
```

如果 PowerShell 禁止运行脚本，先临时放行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

然后一键重建：

```powershell
.\scripts\setup_project.ps1
```

如果只想分步执行：

```powershell
$env:PYTHONPATH = (Get-Location).Path
.\.venv\Scripts\python.exe scripts\sync_public_data.py
.\.venv\Scripts\python.exe scripts\train_conductivity_model.py
.\.venv\Scripts\python.exe scripts\train_lino3_solubility_model.py
```

## 4. 怎么判断训练成功

训练结束后检查这些文件是否存在：

```text
models/conductivity_model.joblib
models/lino3_solubility_model.joblib
models/training_report.json
models/lino3_solubility_report.json
```

也可以直接查看报告：

```powershell
Get-Content .\models\training_report.json
Get-Content .\models\lino3_solubility_report.json
```

电导率模型如果 `r2` 明显大于 0，说明模型至少学到了有效规律。当前版本大约是：

```text
R² ≈ 0.74
MAE ≈ 1.19
```

LiNO3 溶解度模型的 `leave_one_solvent_out_r2_log10` 很低，这是正常但也很重要的信号：数据太少，外推难度很大。所以系统给 LiNO3 结果时会把置信度控制得比较谨慎。

## 5. 本地如何启用完整模型

本地启动脚本已经默认启用完整电导率模型：

```powershell
.\scripts\start.ps1
```

等价于设置：

```powershell
$env:IONMIX_CONDUCTIVITY_MODEL = "enabled"
```

然后启动：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000
```

如果想确认完整电导率模型是否加载，打开：

```text
http://127.0.0.1:8000/api/model-info
```

如果看到：

```json
"available": true
```

说明电导率模型已加载。

## 6. 为什么云端免费版不用完整电导率模型

Render 免费实例内存约 512 MB。完整 `conductivity_model.joblib` 加载后可能超过限制，导致：

```text
502 Bad Gateway
Ran out of memory
```

所以线上免费部署默认是轻量模式：

```text
IONMIX_CONDUCTIVITY_MODEL=disabled
```

轻量模式仍然会使用：

- 分子描述符
- 溶剂化启发式评分
- 传输启发式评分
- LiNO3 溶解度小模型
- 闪点、黏度、安全性、稳定性约束
- 二元/三元溶剂组合搜索

如果升级 Render 到更大内存实例，可以设置：

```text
IONMIX_CONDUCTIVITY_MODEL=enabled
```

然后重新部署。

## 7. 如果我要加入自己的数据，应该加在哪里

### 7.1 加 LiNO3 溶解度数据

当前文件：

```text
data/lino3_solubility.csv
```

需要保持类似字段：

```text
salt
solvent
solvent_name
smiles
temperature_c
solubility_mole_fraction
relative_uncertainty
source
doi
```

如果你新增了 LiNO3 在某个纯溶剂中的溶解度数据，直接追加到这个 CSV，然后运行：

```powershell
.\.venv\Scripts\python.exe scripts\train_lino3_solubility_model.py
```

如果新增溶剂不在 `data/processed/solvent_catalog.csv`，还需要先把它加入 `data/solvent_seed.csv`，再运行：

```powershell
.\.venv\Scripts\python.exe scripts\sync_public_data.py
```

### 7.2 加二元/三元混合溶剂数据

这一步目前还没有完整监督训练管线。现在系统对二元/三元混合物主要是：

```text
纯溶剂标签 + 分子描述符 + 混合规则 + 启发式约束
```

如果你有混合溶剂实测数据，建议新建一个表，例如：

```text
data/mixture_experiments.csv
```

建议字段：

```text
salt
temperature_c
concentration
solvent_a
ratio_a
solvent_b
ratio_b
solvent_c
ratio_c
solubility_observed
conductivity_ms_cm
viscosity_mpas
phase_stable
source
doi
notes
```

有了足够数据后，下一步应该新增一个真正的混合配方模型，例如：

```text
scripts/train_mixture_model.py
models/mixture_model.joblib
```

这个目前还没做，因为没有足够公开混合配方标签。

## 8. 训练数据越多越好吗

不是简单越多越好，关键是覆盖你关心的空间。

对这个项目最有价值的数据是：

- LiNO3 在二元/三元混合溶剂中的实测溶解度
- 不同温度下的溶解度
- 不同盐浓度下的电导率
- 黏度和温度依赖
- 析晶/分层/不稳定的失败样本
- 与电极、隔膜、集流体兼容性的实验结果
- 循环性能或库伦效率

尤其是失败样本很有价值。只有成功配方会让模型过度乐观。

## 9. 当前训练体系的边界

当前系统适合回答：

```text
哪些溶剂组合值得先做实验？
大概从什么配比开始？
这个建议主要靠哪些性质支撑？
置信度为什么高或低？
```

当前系统不适合直接回答：

```text
这个配方一定能溶解多少 LiNO3？
这个配方就是最佳电解液？
这个配方一定能用于某个具体电池体系？
```

最终必须实验确认：

- 实际溶解度
- 长期析晶
- 电导率
- 黏度
- 电化学窗口
- 安全性
- 电池循环表现

## 10. 一句话理解训练逻辑

当前训练逻辑是：

```text
公开实验数据 → 清洗与标准化 → 构造分子/配方特征 → 训练性质模型 → 用验证集估计误差 → 在网页中结合约束和权重排序候选配方
```

所以，`setup_project.ps1` 是“把这一套流程跑完”；真正决定模型上限的是你能不能继续补充高质量实验数据。
